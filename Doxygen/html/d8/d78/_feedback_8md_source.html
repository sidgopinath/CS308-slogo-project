2 ====
3 * Some functionality missing such as being able to control turtle in multiple ways, a user-configurable workspace, and saving and loading sessions and preferences.
4 * Break functionality code out of view and put it into controller. Code interacting between model and view should not interact so deeply. View should be displaying information and passing information back to the model without caring how it is being processed or what is happening to that data.
5 * Remove System.out.println debug code before committing final submission.
6 * Avoid hard-coded strings, even when accessing Map values. They aren't compiler-checked, so a simple typo can blow everything up at Runtime, and if you want to change a name later it's nearly impossible to do so.
7 * I understand your reasoning behind the switch statements, but I think it could be done more cleanly. Also never hard-code strings, especially when they're used for control.
